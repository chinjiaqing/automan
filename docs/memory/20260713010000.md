## 完成事项

### 1. 运行配置（WorkflowRunConfig）

新增工作流运行配置体系，支持触发方式、定时调度、停止条件：

**类型定义（shared/types.ts）：**
- `WorkflowTriggerMode`: `'immediate' | 'scheduled'`
- `ScheduleTime`: `{ hour: number, minute: number }`
- `WorkflowRunConfig`: 完整运行配置（触发模式、定时时间、成功/失败上限）
- `FlowState`: `'idle' | 'pending' | 'processing' | 'success' | 'fail' | 'completed'`

**数据库（server/db/schema.ts）：**
- 新增 `workflowRunConfigs` 表：triggerMode, scheduleTimes(JSON), maxSuccessCount, maxFailCount

**API 路由（server/routes/workflow.routes.ts）：**
- `POST /api/workflows/run-config` — 保存运行配置
- `GET /api/workflows/run-config/:deviceId/:workflowId` — 获取单个配置
- `GET /api/workflows/run-configs/:deviceId` — 获取设备所有配置

**前端 API（web/api/workflow.ts）：**
- `saveRunConfig` / `getRunConfig` / `getRunConfigs`

### 2. CronManager 定时调度

新增 `server/modules/workflow/cron.manager.ts`：
- 基于 `node-cron` 的动态 cron 管理
- `register(deviceId, workflowId, times, callback)` — 注册定时任务
- `unregister(deviceId, workflowId)` — 注销单个
- `unregisterAll()` — 注销全部
- compositeKey（`deviceId:workflowId`）缓存管理

### 3. WorkflowActor 状态机

`workflow.actor.ts` 全面重写，引入 6 态状态机 + 计数 + 自动停止：

**状态流转：**
```
idle → pending → processing → success/fail → idle (循环)
                             → completed (达到上限，终态)
```

**关键机制：**
- `markPending()` — cron 回调设置 pending，等待下次截图
- `processResult()` — endSuccess → successCount++, endFail → failCount++, end → 不计数
- 达到上限自动进入 completed 终态，注销 cron、取消截图订阅
- `autoPending()` — 定时模式下执行完毕后自动回到 pending，确保下次截图继续执行（修复 neutral end 卡死问题）

**三值成功语义：**
- `end` 节点 → `success: undefined`（中性，不计数）
- `endSuccess` 节点 → `success: true`
- `endFail` 节点 → `success: false`

## 关键决策

1. **三值成功语义**：`end` 节点返回 `undefined`（中性不计数），只有显式的 `endSuccess`/`endFail` 参与计数
2. **autoPending 机制**：定时模式下每次执行完毕自动回到 pending，确保截图驱动的连续执行循环
3. **CronManager 单例 + compositeKey**：用 `deviceId:workflowId` 作为缓存键，支持多设备多工作流独立调度
4. **坐标转换用 getBoundingClientRect**：精确获取图片实际显示区域，自动处理 letterbox 偏移，比手动计算 sx/sy 更可靠
5. **DatePicker timeOnly**：替代 InputNumber 时间输入，提供更好的 UX
