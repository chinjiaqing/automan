# 2026-07-09 工作总结 — 找图核心封装 + CanvasView 重构

## 完成事项

### 1. 找图核心 API 封装（core/find-pic.ts）
- 新建 `server/src/core/find-pic.ts`，对外暴露稳定的 `findPic()` 函数
- 签名：`findPic(options: FindPicOptions) => Promise<FindPicResult>`
- 支持 region 参数 `[x1,y1,x2,y2]`，全0表示全图搜索
- 底层通过 Python + OpenCV 实现，但接口与实现完全解耦
- HTTP 路由和脚本开发者共用同一个 findPic 函数

### 2. 底层实现切换：Python + OpenCV
- 新建 `server/src/modules/device/find_pic.py`，使用 `cv2.matchTemplate`（TM_CCOEFF_NORMED，FFT 加速）
- 删除旧的 `dm.service.ts`（NCC + SAT 算法，性能差）
- 卸载 `opencv-wasm`（WASM 版本慢，3.7s/次）
- 性能：1280x720 全图搜索 ~400ms（含进程启动），区域搜索 ~200ms

### 3. shared types 增加 region 参数
- `FindPicRequest` 新增 `region?: [number, number, number, number]`

## 关键决策

1. **Python + OpenCV 而非 JS 实现**：JS 无法在合理时间内完成像素级模板匹配，Python 原生 C++ OpenCV 通过 FFT 加速，性能提升 60+ 倍
2. **core/ 层封装稳定接口**：`findPic()` 函数签名不变，底层实现可随时替换（Python / dm.dll / 其他）
3. **CanvasView 拆分**：遵循"单文件≤300行"规范，按职责拆分为 composable + 子组件
