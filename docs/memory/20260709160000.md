# 2026-07-09 工作总结 — libs 标准化重构

## 完成事项

1. **libs 目录标准化**
   - 将 Python 脚本（find_pic.py）和 TS 封装统一收敛到 `server/src/libs/`
   - 删除旧的 `server/libs/`（Python 资源）和 `core/find-pic.ts`（位置不对）
   - 目录结构：`libs/index.ts` + `find-pic.ts` + `python-helper.ts` + `find_pic.py` + `requirements.txt`

2. **python-helper.ts 独立拆分**
   - 从 index.ts 中提取 Python 进程调用逻辑为独立文件
   - 提供 `getPythonPath()` / `getScriptPath()` / `runPythonScript()` 三个通用工具
   - 新增 Python 功能时只需调用这些工具

3. **index.ts 统一导出**
   - `export { findPic }` 作为唯一入口
   - 同时导出 Python 工具函数供后续扩展

4. **.venv 虚拟环境隔离**
   - Python 依赖安装到 `server/.venv/`，不污染系统 Python
   - 用户清理只需删除 `.venv/` 目录

5. **文档同步**
   - 产品文档：技术栈新增 Python，项目结构新增 `bin/` `.venv/` `libs/`
   - 开发前必读：新增「Python 集成规范」章节

## 关键决策

- **src/libs/ vs server/libs/**：选 `src/libs/` 因为 tsconfig rootDir=src，libs/ 在 src 内才能被 TS 编译
- **python-helper 独立文件**：通用 Python 调用工具，新增 OCR 等功能时可直接复用
- **.venv 在 server/ 根目录**：与 tsconfig 的 rootDir 解耦，分发时可选择性打包
