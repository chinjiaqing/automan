# 桌面端（Electron）v1 落地

## 本次完成

将项目包装为 Electron 桌面应用（Windows 为主、macOS 为辅），新增第四个 workspace 包 `desktop/`。mac 侧全链路已验证（dmg 出包 + packaged 冒烟通过），Windows 侧脚本与 CI 已就绪待真机验证。

### 架构决策（关键）

**server 不进 Electron 进程（含 utilityProcess），用随包便携 Node 跑 sidecar。**
理由：better-sqlite3 非 N-API 且无官方 Electron ABI 预编译产物；`ELECTRON_RUN_AS_NODE`/utilityProcess 同样是 Electron ABI，逃不掉重编译；重编译会污染 pnpm 共享 store，破坏 tsx 开发模式与 start.bat 网页版。便携 Node（复制构建机 `process.execPath`，改名 `automan-server(.exe)`）让三种分发形态共用同一套 Node-ABI 产物，且 sidecar 可脱离 Electron 独立冒烟。

### 踩坑记录（后来者注意）

1. **electron-builder 在遍历层硬排除 node_modules**，extraResources 的 filter 覆盖无效 → 用 `afterPack` 钩子手动补拷 `resources/server/node_modules`
2. **pnpm 10 deploy** 默认要求 injected workspace → `--legacy`；产物默认 .pnpm symlink 布局 → `--config.node-linker=hoisted` 得到扁平真实目录；目标目录必须为空
3. **原生模块 ABI**：prebuild 与 `pnpm install` 时的 Node 绑定，便携运行时必须同版本（prepare-node 复制 execPath 天然对齐；CI 用同一个 setup-node）
4. **sidecar env 必须带 `NODE_ENV=production`**：它同时门控静态托管注册与 pino-pretty transport（devDep，不在 bundle 里），漏了 = 白屏 + 启动即崩
5. **package.json 需要 `productName`**，否则打包后 userData 目录名是 `@automan/desktop`
6. electron 二进制 postinstall 下载在国内常静默失败 → `ELECTRON_MIRROR` + 重跑 `node_modules/electron/install.js`
