# 2026-07-13 工作总结 — 工作流状态机修复 + 截屏智能启停 + 暂停功能

## 背景

定时工作流达到成功/失败上限后进入 `completed` 状态，原实现会级联销毁 CronJob + WorkflowActor + ScreenshotDispatcher + DeviceSession，导致：
1. 次日定时任务无法再次触发
2. 所有工作流达标后设备自动停止
3. 无法手动暂停/恢复设备

## 完成事项

### 1. Cron 生命周期与 completed 状态解耦

**问题**：`handleAutoCompleted` 监听 `WORKFLOW_STATE` 事件，检测到 `completed` 后调用 `stopWorkflow` 销毁一切。

**方案**：
- 删除 `handleAutoCompleted` 事件监听和方法
- `completed` 语义不变（"本轮已完成"），但不再触发资源销毁
- Cron 生命周期严格绑定设备级控制（用户显式开始/停止）

**文件**：`server/src/modules/workflow/service.ts`

### 2. Cron 回调支持 completed 重入

**问题**：`markPending` 仅接受 `idle` 状态，`completed` 时被忽略，次日 cron 触发无效。

**方案**：
- `markPending` 新增 `completed` 分支：重置 `successCount` / `failCount` / `cancelled`，设置 `flowState = 'pending'`
- 定时工作流每日自动重触发，无需人工干预

**文件**：`server/src/modules/workflow/workflow.actor.ts`

### 3. 截屏智能启停（syncDispatcher）

**问题**：所有工作流 completed 后截图调度器仍在运行，每 2s 执行 ADB 截屏 + sharp 压缩，浪费资源。

**方案**：
- `ScreenshotDispatcher` 新增 `pause()` / `resume()` 方法（保留 entry，仅控制定时器）
- `WorkflowService` 新增 `syncDispatcher(deviceId)` 作为截图启停的**唯一决策点**
- 通过 EventBus 监听 `WORKFLOW_STATE` 事件自动驱动，无需在多处分散逻辑

```
syncDispatcher(deviceId):
  ├─ pausedDevices.has → return（手动暂停优先）
  ├─ 有非 completed actor → dispatcher.resume
  └─ 全 completed → dispatcher.pause
```

**文件**：`server/src/modules/workflow/screenshot.dispatcher.ts`、`server/src/modules/workflow/service.ts`

### 4. 设备级暂停/恢复功能

**需求**：用户可手动暂停设备截图，恢复后自动衔接。

**方案**：
- `DeviceRunStatus` 枚举新增 `PAUSED`（设备显式状态，前端可展示）
- `pausedDevices: Set<string>` 跟踪手动暂停设备
- `pauseDevice(deviceId)` / `resumeDevice(deviceId)` 统一入口
- `resolveDeviceStatus()` 优先级：手动暂停 > 错误 > 运行中
- `onScreenshot` 增加 `completed || cancelled` 前置拦截（双保险）

**API**：
- `POST /api/workflows/pause` — 暂停设备截图
- `POST /api/workflows/resume` — 恢复设备截图

**前端**：
- LogPanel 新增暂停/恢复按钮
- HomeView 设备图标适配 PAUSED 状态（蓝色暂停图标）

**文件**：`shared/src/types.ts`、`server/src/routes/workflow.routes.ts`、`web/src/api/workflow.ts`、`web/src/composables/useWorkflowRun.ts`、`web/src/components/LogPanel.vue`、`web/src/views/HomeView.vue`

## 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| completed 状态处理 | 保留语义，不销毁资源 | 最小改动（~10 行），cron 回调自然重入 |
| 截图启停驱动方式 | EventBus WORKFLOW_STATE 事件 | 统一决策点，避免逻辑分散 |
| 暂停实现层级 | 设备级（ScreenshotDispatcher） | 截图是唯一触发源，暂停截图 = 全局暂停 |
| 暂停/自动优先级 | 手动暂停 > 自动启停 | `pausedDevices` 拦截 `syncDispatcher` |
| 替代方案（已否决） | `limitReached` 改名 + 三维状态机 | 过度设计，6 文件 100+ 行改动 |

## 行为对照表

| 场景 | 改动前 | 改动后 |
|------|--------|--------|
| 定时 9:00，成功 2 次达标 | completed → cron 注销 + actor 销毁 → 次日不触发 | completed → cron/actor 保留 → 次日自动重置执行 |
| 所有工作流达标 | 设备自动停止 | 截图自动暂停，设备保持 RUNNING |
| 用户手动暂停 | 不支持 | 截图停止，设备显示 PAUSED |
| 用户手动恢复 | 不支持 | 截图恢复，自动衔接 |
| 立即模式达标 | 销毁一切 | actor 存活但静默，等用户手动停止 |
