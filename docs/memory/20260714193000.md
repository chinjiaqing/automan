# 2026-07-14 工作总结 — 工作流变量解析修复 + AI 生成约束

## 背景

用户运行"启动应用"工作流时遇到两个问题：

1. **外部输入包名不生效**：`launchApp` / `appRunning` 等应用类节点直接把配置字面量 `{{variable_input_pkg.value}}` 传给 adb，未解析 `{{ }}` 引用。
2. **loop 直接判失败**：loop / condition 节点的 `left` 写成裸 `session.detectCount`（无花括号），引擎无法解析 → 条件恒为假 → 立即走 exit → `endFail`，流程未进入轮询。

经排查，根因是应用类节点不解析变量引用、变量池被强制 `Number()` 成数字、以及 AI 生成脚本使用了引擎不支持的裸 `scope.name` 格式（源头数据异常）。

## 完成事项

### 1. 应用类节点变量引用解析（根因一）

**问题**：`engine.ts` 的 `launchApp` / `killApp` / `restartApp` / `appRunning` 四节点（main 块 + 循环 inner 块共 8 处）直接读 `node.config.packageName`，未走 `resolveValue`，导致 `{{ }}` 字面量原样进入 adb。

**修复**：统一改为 `cleanPkgName(resolveValue(node.config.packageName, ctx.outputs, ctx.variables))`，与其它节点一致。

**文件**：`server/src/modules/workflow/engine.ts`

### 2. 变量类型保留（根因二）

**问题**：`execVariable` 对所有变量 `Number()` 强转，`initVariables` 默认值也用 `Number(value) || 0`，外部输入的字符串包名被转成 `0`。

**修复**：
- `execVariable`：`createOrSet` 已存在时保留当前值原类型；`set` 直接存解析结果；`add/sub/mul/div` 仍走数字运算。
- `initVariables`：默认值改为 `value !== undefined && value !== '' ? value : 0`，字符串包名原样保留，空值回落 `0`（不破坏数字变量语义）。

**文件**：`server/src/modules/workflow/engine.ts`、`server/src/modules/workflow/workflow.actor.ts`

### 3. AI 生成 flow 变量引用约束（提示词 + 引擎）

**问题**：AI 生成的 flow 使用裸 `session.detectCount`，引擎不解析。需在源头约束 AI 使用带花括号的规范格式，并让引擎支持该格式。

**修复**：
- `ai.routes.ts` 系统提示词规则 7 新增约束：变量引用一律使用 `{{scope.varName}}` 形式（如 `{{session.detectCount}}`、`{{input.packageName}}`），**严禁裸 `scope.varName`**。
- `refResolver.ts` 新增 `{{scope.name}}` 解析（scope ∈ `session/local/input/global`）→ `variables[name]`，与之前废弃的裸 `scope.name` 区分（后者仍不解析）。

**文件**：`server/src/routes/ai.routes.ts`、`server/src/modules/workflow/refResolver.ts`

## 关键决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 应用节点解析方式 | `resolveValue` 包裹 | 与其它节点统一，复用现有引用解析逻辑 |
| 变量类型 | 保留原类型，算术操作仍数字 | 包名等字符串不被破坏，计数器照常工作 |
| AI 约束格式 | `{{scope.name}}` 带花括号 | 区别于废弃的裸 `scope.name`，引擎可明确解析 |
| 引擎支持范围 | 仅 `{session,local,input,global}` 前缀 | 避免误伤 `com.xxx` 等含点的合法字符串 |

## 验证

- `resolveValue('{{session.detectCount}}', {}, {detectCount:0})` → `0`
- `resolveValue('{{input.packageName}}', {}, {packageName:'com.x.live'})` → `'com.x.live'`
- `resolveValue('session.detectCount', ...)` → 原样返回（被约束禁止的格式，符合预期）
- `tsc --noEmit` 通过，0 lint 错误

## 遗留 / 下一步

- 字符串变量参与 `add/sub/mul/div` 等算术操作会得到 `NaN`/`0`，目前由节点语义保证不混用；如需更严格可在配置面板做类型标注。
- 既有裸 `scope.name` 的历史工作流数据需在源头修正为 `{{scope.name}}`（引擎不向后兼容裸格式，符合设计）。
