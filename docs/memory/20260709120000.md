## 技术方案说明

### 为什么不用 dm.dll 的 findPic？

dm.dll 的 `findPic` 内部调用 `dm.Capture` 抓取**实际屏幕**，无法对 ADB 截图的内存图片做匹配。因此改用 sharp（Node.js 高性能图像处理库）+ NCC 算法纯服务端实现，效果等价且更可靠。

### NCC 算法 + SAT 加速

- **NCC（归一化互相关）**：对光照变化鲁棒，适合游戏截图匹配
- **SAT（Summed Area Table）**：将源图均值/方差计算从 O(tw*th) 降到 O(1)，整体性能提升数十倍
- **去重策略**：距离小于模板尺寸一半的匹配只保留最高分

## 前端交互细节

### Save 按钮
- 位置：框选坐标区域下方，仅有选区时显示
- 功能：用 Canvas 从原图裁剪选区，触发浏览器下载 PNG
- 文件名：`selection_x1_y1_x2_y2.png`

### 找图 Tab
- 上传模板图片后显示预览缩略图 + 移除按钮
- 相似度滑块：30%-100%，默认 80%，实时显示百分比
- 查找按钮：禁用条件为无截图/无模板/查找中
- 结果列表：显示匹配数量 + 耗时，点击可高亮定位
- 截图覆盖层：绿色边框 + 半透明填充 + 编号/置信度标签
