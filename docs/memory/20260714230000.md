# 2026-07-14 工作总结 — 片段（Fragment）系统完整实现

## 背景

片段是可复用的工作流子流程（函数），支持输入输出参数声明、内部变量操作、被 call 节点调用。本次实现了完整的片段编辑器和引擎集成。

## 完成事项

### 1. 片段数据模型

- `FragmentParam` 接口：`name`（随机生成变量名如 `input_ax1jak`）、`type`、`defaultValue`（string）、`label`（中文标签如"入参1"）
- `Fragment` 接口：`id`、`name`、`groupId`、`inputs`、`outputs`、`nodes`、`edges`
- 数据库 `fragments` / `fragment_groups` 表（Drizzle schema + 幂等 migrate）

**文件**：`shared/src/types.ts`、`server/src/db/schema.ts`、`server/src/db/migrate.ts`

### 2. 片段 IO 编辑器（FragmentIOEditor.vue）

- 左右两栏布局：输入参数（左）、输出参数（右）
- 每行包含：label 输入框（默认"入参1"/"出参1"）、变量名（随机6位字符，灰色 monospace 小字显示）、默认值输入、类型选择、删除按钮
- 新建参数时自动生成变量名和默认 label

**文件**：`web/src/components/flow/FragmentIOEditor.vue`

### 3. "声明即变量"模式（核心设计）

- 声明的输出变量在片段开始执行时注入 `ctx.variables[output.name] = defaultValue`
- 内部 variable 节点可以写入这些 output 变量
- 片段结束时引擎自动从 `ctx.variables` 提取输出值返回给调用方
- 不同分支可以给同一个 output 赋不同值（条件分支覆盖）
- 返回值数量和类型固定，只有值可变

**文件**：`server/src/modules/workflow/engine.ts`（execCall 方法）

### 4. 片段编辑器（FragmentView.vue）

- 四栏布局：分组列表 | 片段列表 | 画布（含 IO 编辑器 + VueFlow） | 配置面板
- 分组管理：新建/重命名/删除
- 片段管理：新建/编辑/删除，按分组筛选
- 节点面板：排除 call/endSuccess/endFail/end/return 节点
- 作用域限制：fragmentMode 下隐藏 scope 字段，强制为"本轮"（local）
- 数据源下拉：分组展示入参、出参、变量节点（Teleport 到 body 避免 overflow 裁剪）

**文件**：`web/src/views/FragmentView.vue`

### 5. 数据源下拉组件（DataSourceSelect.vue）

- 分组展示：入参（`__input`）| 出参（`__output`）| 变量节点 | 新建
- 使用 Teleport 渲染到 body，`position: fixed` 定位，避免被 ConfigPanel overflow 裁剪
- 点击外部自动关闭
- 选项显示 label（中文名）+ 变量名（monospace 灰色）

**文件**：`web/src/components/flow/DataSourceSelect.vue`

### 6. 引擎片段调用（execCall）

- 变量隔离：快照 → 注入 output 默认值 → 注入 input 参数 → 执行内部节点 → 提取输出值 → 恢复父级
- 自然结束时从 `ctx.variables` 提取声明的 output 变量值
- return 节点仍保留兼容（从 node.config 收集输出值）
- 支持所有节点类型在片段内执行

**文件**：`server/src/modules/workflow/engine.ts`

### 7. 路由与导航

- `/fragments` 路由指向 FragmentView
- 顶部导航"片段"菜单项增加 BETA 标识

**文件**：`web/src/router/index.ts`、`web/src/components/AppHeader.vue`

### 8. ConfigPanel fragmentMode

- 新增 `fragmentMode` prop，为 true 时隐藏 scope 字段
- FragmentView 传入 `fragment-mode="true"`

**文件**：`web/src/components/flow/ConfigPanel.vue`

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| output 关联方式 | 声明即变量 | 无需专用 output 节点，复用 variable 节点写入 |
| 变量名生成 | 随机6位字符 | 避免多片段变量名冲突 |
| defaultValue 类型 | 统一 string | InputText v-model 兼容，引擎按 type 转换 |
| 片段内 scope | 强制 local | 片段是纯函数，无全局/输入变量概念 |
| 片段内 end/return | 移除（从 NodePalette 排除） | 自然结束即可，引擎自动提取 output 值 |
| 下拉框定位 | Teleport + fixed | 避免 overflow-y:auto 裁剪 |
