## 关键架构决策

1. **WS 三层分离**：路由层保持薄层（约 15 行），所有逻辑委托给 Gateway / Dispatcher，严格遵守"WS 不写在 route 里"约束
2. **延迟注入**：Dispatcher 通过 `setActorManager()` / `setTaskService()` 延迟绑定，避免循环依赖
3. **EventBus 广播**：Actor 日志通过 `eventBus → app.wsGateway.broadcast()` 自动推送所有 WS 客户端
4. **Node16 模块解析**：所有相对导入使用 `.js` 扩展名，符合 ESM 规范
5. **`@automan/shared` workspace 协议**：server 通过 `workspace:*` 引用共享包
