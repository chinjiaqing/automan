# 工作总结 — 设备绑定流程修正

## 修正内容

原流程有误：直接让用户手动填 instanceIndex 再绑定，没有先查询可用实例。

**修正后的正确流程：**
1. 用户填写 ldconsole.exe 路径
2. 点击「🔍 查询可用实例」→ `GET /api/devices?path=...` → 返回解析后的实例列表
3. 下拉框自动填充，显示每个实例的 `[index] name — 运行状态`
4. 用户选择实例，别名自动填充
5. 点击「🔗 绑定选中实例」→ 完成绑定，设备 ID 自动填入

## 变更文件

| 文件 | 变更 |
|------|------|
| `shared/src/types.ts` | 新增 `LDInstanceInfo` 类型（index/name/running/pid 等） |
| `server/src/modules/device/ldplayer.service.ts` | 新增 `listInstancesParsed()` 方法，返回结构化实例数组 |
| `server/src/modules/device/device.service.ts` | `listInstances()` 改为返回 `LDInstanceInfo[]` |
| `server/src/app.ts` | `GET /api/devices` 改为接收 `path` query 参数，返回 `{ devices, instances }` |
| `web/index.html` | 设备面板重做：查询→下拉选择→绑定，新增 select 样式 |

## API 变更

| 旧 | 新 |
|----|----|
| `GET /api/devices` → 仅返回已绑定设备 | `GET /api/devices?path=...` → 返回已绑定设备 + 解析后的实例列表 |
| `POST /api/devices/instances` → 返回原始字符串 | `POST /api/devices/instances` → 返回 `LDInstanceInfo[]` |
