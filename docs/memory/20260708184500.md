## API 路由清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/devices` | 列出所有已绑定设备 |
| POST | `/api/devices/bind` | 绑定设备（传入 ldconsolePath + instanceIndex） |
| DELETE | `/api/devices/:id` | 解绑设备 |
| POST | `/api/devices/:id/launch` | 启动模拟器 |
| POST | `/api/devices/:id/quit` | 关闭模拟器 |
| POST | `/api/devices/:id/reboot` | 重启模拟器 |
| POST | `/api/devices/:id/refresh` | 刷新设备运行状态 |
| POST | `/api/devices/instances` | 查询 ldconsole list2 原始输出 |

## 关键设计决策

1. **ldconsolePath 由用户传入**，不自动探测本机路径，适配任意部署环境
2. **绑定时验证路径**：调用 `ldconsole list2` 确认路径可用
3. **HTTP 控制 + WS 推送**：设备控制用 REST API，状态变更未来通过 WS 广播
4. **Device = 一个模拟器实例**：1 Device 跑 1 个游戏，游戏内多个 Task 串行执行
