# Automan 产品文档 Part 1 — 已实现功能

> 本文档仅描述**已实现并可用**的功能，未实现的需求不在本文档范围内。

---

## 1. 产品定位

Automan 是一个面向模拟器自动化任务的控制面板。用户通过 Web 页面管理多台模拟器设备，使用开发板进行截图/圈图辅助操作。

**当前核心能力：**
- 多设备管理（CRUD，绑定雷电模拟器实例）
- WebSocket 实时连接（连接即登录，断开即退出）
- 开发板工具（设备截屏、框选坐标、缩放调整、找图匹配）

---

## 2. 核心概念

### 2.1 设备（Device）

设备是一个**任务合集入口**，绑定一台模拟器来运行。

| 属性 | 说明 |
|------|------|
| 别名 | 用户自定义名称，如"大号专用"、"小号专用" |
| 模拟器路径 | ldconsole.exe 文件的绝对路径 |
| 实例序号 | 雷电模拟器的 instance index |
| 状态 | **运行中** / **停止**（仅此两种） |

### 2.2 连接（Connection）

- **登录 = WebSocket 连接成功**
- 输入服务器地址（默认 `127.0.0.1`）和端口（默认 `3000`）
- WS 连接成功 → 自动跳转首页
- WS 断开 → 自动退出到登录页
- 已连接用户访问登录页 → 自动重定向到首页

## 4. 已实现页面

### 4.1 登录页（LoginView）

```
┌──────────────────────────────────────┐
│                                      │
│        ⚡ Automan                    │
│                                      │
│   服务器地址  [127.0.0.1      ]      │
│   端口        [3000            ]     │
│                                      │
│        [ 连接 ]                      │
│                                      │
└──────────────────────────────────────┘
```

- 全屏独立布局，不经过 AppLayout
- WS 连接成功后跳转首页
- 连接失败显示错误提示

### 4.2 首页 — 设备与工作流运行控制（HomeView）

```
┌─ 设备列表 ─┬─ 工作流列表 + 配置 ─┬─ 操作按钮 (通屏) ──────────┐
│  设备 A ● │  ☑ 工作流1          │ [开始] [停止]  已运行 00:05 │
│  设备 B   │  ☐ 工作流2          ├─ 实时看板 ─────────────────┤
│            │  ☑ 工作流3          │  [截图 + 识别框/点击波纹]    │
│            │  ── 运行配置 ──     │  (画布点击可触发 ADB 点击)   │
│            │  触发: ●立即 ○定时  ├─ 日志滚动区 ────────────────┤
│            │  定时: [09:00] [+]  │  07-13 00:21 [info] ...      │
│            │  成功上限: [2]      │                              │
│            │  失败上限: [2]      │                              │
│            │  [保存设置]         │                              │
└────────────┴────────────────────┴──────────────────────────────┘
```

**左侧设备列表：**
- 设备名称 + 运行状态指示器（● 绿色=运行中、● 橙色=空闲、● 红色=错误、○ 灰色=未运行）
- 默认自动选中第一个设备
- 勾选快照自动保存到 DB，刷新页面自动恢复

**中间工作流 + 配置区：**
- 工作流勾选开关
- **运行配置表单**：
  - 触发方式：立即（每次截图自动执行）/ 定时（cron 指定时间点触发）
  - 定时时间：DatePicker 时间选择器（timeOnly 模式），支持多个时间点
  - 成功 N 次后停止 / 失败 N 次后停止（0=不限制）
  - 保存设置按钮（Toast 反馈保存成功/失败）
- **FlowState 实时展示**：空闲 / 等待 / 执行中 / 成功 / 失败 / 已完成 + 计数显示

**右侧实时看板 + 日志：**
- 实时看板：截图 + 注解叠加（识别框、点击波纹、文字区域）
- **画布点击**：点击看板图片触发模拟器 ADB 点击，自动处理 object-fit:contain 的 letterbox 偏移
- 点击反馈波纹动画
- 左上角提示文案：“每2秒刷新” / “点击图片和操作模拟器”
- 日志滚动区：实时显示工作流执行日志

### 4.3 开发板（CanvasView）

> 路由：`/track`，name: `track`

```
┌─ 操作区 (w-60) ─┐ ┌─ 截屏区 (flex-1) ──────────────────┐
│                  │ │                                      │
│ 当前设备 [▼]     │ │                                      │
│                  │ │          设备截图显示                  │
│ [ 截屏 ]         │ │          鼠标拖拽框选                  │
│                  │ │          顶点等比缩放                  │
│ 当前选中：  📋 ✕ │ │          边线单向缩放                  │
│ [x1,y1,x2,y2]   │ │          拖动移动选区                  │
│ 宽 W × 高 H     │ │                                      │
│                  │ │                                      │
│ [找图] [OCR]     │ │                                      │
│  ...             │ │                                      │
│ ────────────── │ │                                      │
│ [点击] [范围点击]│ │                                      │
│  [0,0]  [点击]  │ │                                      │
└──────────────────┘ └──────────────────────────────────────┘
```

**设备选择：**
- 下拉框，默认自动选中第一个设备

**截屏：**
- 点击「截屏」按钮，调用 ADB screencap
- 截图显示在右侧区域

**框选交互：**

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 新建框选 | 鼠标在截图区域拖拽 | 创建新的选区 |
| 移动选区 | 鼠标在选区内部拖拽 | 移动整个选区，实时更新坐标 |
| 等比缩放 | 拖拽四角圆形手柄 | 保持宽高比缩放 |
| 单向缩放 | 拖拽四边矩形手柄 | 仅改变宽度或高度 |
| 清除选区 | 点击选区外空白区域 | 或点击清除按钮（✕） |

**坐标回显：**
- 常显，不闪烁
- 默认 `[0,0,0,0]` 表示全屏，显示图片原始尺寸 + "(全屏)"标记
- 有选区时显示实际坐标 `[x1,y1,x2,y2]` + 选区尺寸
- 坐标基于原始分辨率换算（非显示像素）
- 复制按钮：点击后图标变 ✓（绿色），1 秒后恢复
- 清除按钮：有选区时显示，点击恢复 `[0,0,0,0]`

**视觉反馈：**
- 聚光灯蒙层：选区外半透明黑色遮罩
- 蓝色边框 + 白色 outline + 四角标记
- 实时宽高标注：黑底白字，上边线（宽）+ 右边线（高）
- 贴边吸附：拖动距边缘 < 5px 自动贴边

**Tab 区域：**
- 找图：上传模板图片 → 调整相似度 → 在截图中搜索匹配位置，支持区域搜索（框选区域即为搜索范围）
- OCR：识字模式（识别选区内所有文字）+ 找字模式（在截图中查找目标文字并定位），支持颜色过滤与颜色偏差

**点击区域（跟随 Tab 区域下方）：**
- 点击：输入 `[x, y]` 坐标，通过 ADB 执行单次点击
- 范围点击：输入 `[x1, y1, x2, y2]` 矩形区域，在区域内随机取点执行点击
- 参数格式与找图 `region` 保持一致（数组形式）

```
┌─ 添加设备 ──────────────────────────────────────┐
│                                                  │
│  设备别名                                        │
│  [                    ]                          │

│  │  📁 Program Files                       │     │
│  └─────────────────────────────────────────┘     │
│                                                  │
│  选择实例                                        │
│  ○ [0] 雷电模拟器-0 — 运行中                    │

│                                                  │

```

**交互流程：**
1. 输入设备别名
2. 点击「浏览」→ 打开文件浏览器，逐层导航选择 `ldconsole.exe`
3. 选中后自动加载模拟器实例列表（调用 `ldconsole list2`）
4. 单选目标实例（显示名称 + 运行状态）
5. 确认创建

## 6. API 路由清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/devices` | 查询设备列表 |
| POST | `/api/devices/create` | 创建设备 |
| POST | `/api/devices/delete` | 删除设备 |
| POST | `/api/devices/update` | 重命名设备 |
| POST | `/api/devices/instances` | 查询模拟器实例列表 |
| POST | `/api/devices/screenshot` | 设备截屏（ADB screencap） |
| POST | `/api/devices/find-pic` | 找图（模板匹配，OpenCV） |
| POST | `/api/devices/ocr-words` | OCR 识字（RapidOCR） |
| POST | `/api/devices/ocr-find-str` | OCR 找字（滑动窗口 + Levenshtein 模糊匹配） |
| POST | `/api/devices/click` | ADB 单点点击 |
| POST | `/api/devices/area-click` | ADB 范围随机点击 |
| POST | `/api/workflows/run` | 启动单个工作流 |
| POST | `/api/workflows/run-batch` | 批量启动（设备级） |
| POST | `/api/workflows/stop` | 停止工作流 |
| POST | `/api/workflows/stop-all` | 停止所有工作流 |
| GET | `/api/workflows/running` | 查询运行中工作流 |
| POST | `/api/workflows/run-config` | 保存运行配置 |
| GET | `/api/workflows/run-config/:deviceId/:workflowId` | 获取单个运行配置 |
| GET | `/api/workflows/run-configs/:deviceId` | 获取设备所有运行配置 |
| GET | `/api/filesystem/browse?path=` | 浏览本地文件系统 |
| WS | `/ws` | WebSocket 连接 |

**统一响应格式：**
```typescript
// 成功
{ success: true, data: T }

// 失败
{ success: false, code: 'ERROR_CODE', message: '...' }
```

| 层级 | 技术 |

## 8. 项目结构

```
automan/
├── .bin/            # 分发运行时（start.bat 自动下载，不提交 git）
│   ├── node/        # Node.js 便携版
│   └── python/      # Python embeddable
├── start.bat        # 一键启动（自动下载运行时 + 安装依赖 + 启动服务）
├── server/          # 服务端（Fastify）
│   └── src/
│       ├── routes/      # API 路由
│       ├── modules/     # 业务模块（device, ws, actor, task）
│       ├── libs/        # 工具与脚本层（Python 脚本 + TS 封装 + 统一导出）
│       │   ├── index.ts         # 统一导出层
│       │   ├── python-helper.ts # Python 进程调用封装 + PythonWorker
│       │   ├── find-pic.ts      # 找图 API
│       │   ├── ocr.ts           # OCR 文字识别 API
│       │   ├── adb-click.ts     # ADB 点击 API（单点 + 范围随机）
│       │   ├── find_pic.py      # Python 找图脚本（OpenCV）
│       │   ├── ocr.py           # Python OCR 脚本（RapidOCR）
│       │   └── requirements.txt # Python 依赖清单
│       ├── db/          # 数据库（schema, migrate）
│       └── core/        # 核心（logger, event-bus）
├── web/             # 前端（Vue 3）
│   └── src/
│       ├── views/       # 页面（LoginView, HomeView, CanvasView）
│       ├── components/  # 组件（AppHeader, DeviceDialog, FindPicPanel, OcrPanel）
│       ├── composables/ # 状态管理（useWebSocket, useDevices, useSelection）
│       ├── api/         # HTTP 请求封装
│       ├── layouts/     # 布局（AppLayout）
│       └── router/      # 路由 + 守卫
└── shared/          # 前后端共享（types, utils）
    └── src/types.ts
```

## 9. 分发与部署

用户端无需安装 Node.js 和 Python，`start.bat` 自动下载所有运行时到 `.bin/` 目录：

| 运行时 | 位置 | 来源 |
|--------|------|------|
| Node.js v24.14.1 | `.bin/node/` | USTC 镜像 |
| Python 3.12.10 | `.bin/python/` | 华为云镜像 (embeddable) |

**用户首次使用：** 运行 `start.bat` 即可，脚本自动完成环境下载、依赖安装、服务启动。
